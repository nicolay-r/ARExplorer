We consider the following tools:

#1. tool that represents the main contribution of the agent: bulk-ner.
    bulk-ner provides a service of annotating the named entities to the massive volume of texts that solves the context limit problem of LM models.

    It provides the output in the most readable form that represents lists of the tokens and a notation of entities in it.
    We can use this information as an output context, suitable for LLMs.

#2 tool that determins connection between entities.
    It's something that could be done with LLM.
    Given a dataset of pairs, where how we can approach this is to apply the bulk-chain.
    bulk-chain allows to use batching technique of querying LLM (llm-as-service) by implementing adapter to any provider.
    Input: schema, is a custom chain or a single prompt in which parameter is the TYPE OF RELATION
    We can by default register sentiment and analysis type of relations, which assumes that we output. [give link to task / papers / of related studies].
    Classify relations among three different facets, which is positive, negative, neutral.

#3 Graph operations.
    Simple tool that performs Union, Intersection operations.
    Since we did not expect here use of third-party resouces, this could be a single function.
    Therefore, it does not require use for MCP.

Examples:
    Search and filter in massive database of documents.
